Pizza Views¶

Endpoint docs for pizza.views:

pizza.views.close(request, pizza_order_id)[source]¶

This close view get pizza order by id, mark it delivered and redirects to view page of pizza order.

Parameters:request – HttpRequest, pizza_order_id
Returns:
  • HttpResponse with rendered template view.html with new state of order
  • Status code 404 with message 'Does not exist'
  • Status code 405 if request method not GET
pizza.views.create(request)[source]¶

This view renders a form for creating new PizzaOrder (pizza/create.html) or accepts POST requests with the form to save its data.

Parameters:request – HttpRequest, accepted methods: GET, POST
Returns:
  • HttpResponse with rendered view form for new order
  • Status code 405 if request method not GET
pizza.views.index(request)[source]¶

This view renders the main page, listing all orders. Every link in the list points to the order (‘pizza:view’ pizza.id, template view.html)

Parameters:request – HttpRequest
Returns:
  • HttpResponse with rendered view template index.html
  • Status code 405 if request method not GET
pizza.views.stats(request, *args, **kwargs)[source]¶

This view creates an async task to collect various statistics about the created orders. As a result returns an HttpResponse with task’s id.

Parameters:request – HttpRequest, allowed methods: GET
Returns:
  • HttpResponse with task’s id
  • Status code 405 if request method not GET
pizza.views.view(request, pizza_order_id)[source]¶

This view collect data about pizza order by id.

Parameters:request – HttpRequest, pizza_order_id
Returns:
  • HttpResponse with rendered template selected order view.html
  • Status code 405 if request method not GET
